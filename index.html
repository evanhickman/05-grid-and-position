HTML INTRO

Lesson 1

In Lesson 1, we reinforced the fundamentals of HTML and CSS, basic terminal navigation, and the principles of Git. After a tour of the essential Git commands, we learned to open our editor from the terminal and got to work on a broken portfolio page. Emphasized most in this lesson was correct implementation of HTML and CSS syntax.

Beginning with this sort of head on dive into large files with lots of errors, though daunting at first, proved to be the perfect step in getting comfortable with the subtle complexity of HTML and CSS.

BOX MODEL

Lesson 2

In Lesson 2, we took a deeper look into some of the most common HTML and CSS elements and properties. First on the list, create a clean, functioning Nav Bar using floats and display values. Next, a simple grid layout that included margins, borders, and a convenient use for the HTML span. Aside from the HTML and CSS, there was the charge to further explore GitHub and follow some prominent developers.

POSITION & LAYOUT

Lesson 3

In Lesson 3, after an introduction to the Daily Planner App, we reviewed the box model some common display techniques. We look at how to break down a layout into small pieces in order to solve problems individually and efficiently. We also covered a layout method known as the "Jamaican-ized" approach, which separates layout elements into easily recognizeable chunks of content that allow for easier visualisation and placement. In addition, we also talked through using margins effectively (i.e. negative margins), as well using elements intelligently so as to maximize reusability in layout and HTML structure.

FLEXBOX

Lesson 4

Lesson 4 was crucial in understanding advanced use of iTerm and Chrome's dev tools. After touching on these, we moved on to a review of floats before diving into one of the most important concepts covered to date: Flexbox. We covered the most common flex properties and identified appropriate usage for each.

After Flexbox, we moved into Pseudo selectors and their power in creating better UX. We learned last-of-type and nth-child and how to use these sparingly in the pursuit of cleaner code. Finally we had a quick review on units and added two new units, vh and vw, to our wheelhouse.
